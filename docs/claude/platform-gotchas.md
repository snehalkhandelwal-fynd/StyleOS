# Platform Gotchas

> Start empty, add entries as you discover them.
> This is often the highest-value doc in the repo. Every weird bug you hit becomes institutional knowledge.

## How to Add Entries

Each entry should have:
1. **Symptom**: what goes wrong
2. **Cause**: why it happens
3. **Fix**: the workaround or solution

Keep entries concise. If it needs more than a paragraph, link to a longer doc.
